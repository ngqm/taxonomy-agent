\subsection{Evaluation Methodology}
\label{sec:eval-method}

We evaluate \textsc{taxonomy\_agent} on two benchmarks. \textbf{20~Newsgroups}~\citep{lang1995newsgroups} probes whether the orchestrator--judge loop matches established topic-modeling baselines on a corpus with known-good gold structure; we use a 487-document stratified subsample (25 per class, 13 emptied by sklearn's header/footer/quote filter) under five independently sampled random seeds at the install default and three at the Sonnet pair. \textbf{CoT-Pattern} is a 149-item corpus of chain-of-thought traces evaluated under three seeds at the install default and one at the Sonnet pair, injected with five safety-relevant failure modes (sycophantic capitulation, post-hoc rationalization, unfaithful paraphrase, reward-hack verbalization, hallucinated premise), drawn from MATH~\citep{hendrycks2021math} Level~4--5 problems via the public MATH-500 subset, generated by Gemini-3.1-Flash-Lite, and retained only when three independent Claude Opus 4.8 classification passes (different sampling seeds) all agreed on the injected pattern. We compare \textsc{taxonomy\_agent} at the install default (DeepSeek-v4-Flash for both orchestrator and judge) against six baselines. Two non-LLM classical baselines: BERTopic~\citep{grootendorst2022bertopic} at library defaults with a \texttt{min\_topic\_size} sweep, and LDA~\citep{blei2003lda} at $K{=}16$. Three LLM baselines we wrote ourselves to occupy the standard design families: \texttt{single\_shot} prompts one LLM call to both propose and label, \texttt{iterative\_proposal} runs an iterative LLM proposer with label refinement (the TopicGPT~\citep{pham2024topicgpt} family), and \texttt{embed\_cluster\_llm} embeds with all-MiniLM-L6-v2, runs k-means at $K{=}16$, names each cluster with one LLM call, then assigns every item with the cheap judge (the embed-cluster-LLM family popularised by TnT-LLM~\citep{wan2024tntllm}; we do not include TnT-LLM's distilled-classifier phase). One concurrent-work head-to-head: we run the published~\citet{brady2025iterativetax} pipeline (their code, their prompts, all-MiniLM-L6-v2 + HDBSCAN + iterative per-cluster topic synthesis), with the same Llama-3.2-3B model served via OpenRouter to match our uniform OpenRouter routing. We report external metrics (purity, NMI, ARI), intrinsic topic-quality metrics (NPMI, $C_v$ coherence~\citep{rehurek2010gensim}, label redundancy), dollar cost from OpenRouter \texttt{usage.cost}, and wall-clock time. Construct-validity arguments for the single-source synthetic CoT corpus, per-pattern source survey, and the commensurability discussion for purity vs.\ accuracy on CoT appear in Appendix~A.
