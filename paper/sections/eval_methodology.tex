\subsection{Evaluation Methodology}
\label{sec:eval-method}

To evaluate whether \textsc{TaxonomyAgent}'s orchestrator--judge loop
produces taxonomies competitive with established baselines while
remaining cheap to run, we run experiments on two benchmarks.

\paragraph{Datasets.}
\textbf{20~Newsgroups}~\citep{lang1995newsgroups} is our classical
benchmark with established gold structure over 20 topic classes. We
use a 487-document stratified subsample (25 per class; 13 documents
are emptied by sklearn's default header/footer/quote filter and
dropped) under five independently sampled random seeds.
\textbf{DarkBench}~\citep{kran2025darkbench} is a real-data benchmark
whose categories are not surface topics: 660 user prompts eliciting one
of six manipulative dark patterns from an AI assistant, balanced at 110
per category. Each prompt's category is a property of its manipulative
intent and not its vocabulary, and the six categories cut across surface
topic, so a taxonomy that groups by topic does not recover them. We evaluate DarkBench over three seeds with no hint about the
number of categories.

\paragraph{Baselines.}
We compare \textsc{TaxonomyAgent}
(DeepSeek-v4-Flash for both orchestrator and judge) against six
baselines. Two non-LLM classical baselines:
BERTopic~\citep{grootendorst2022bertopic} at library defaults with a
\texttt{min\_topic\_size} sweep, and LDA~\citep{blei2003lda} at
$K{=}16$. Three LLM baselines we wrote to occupy standard design
families: \textsc{Single-Shot} prompts one LLM call to both propose
and label, \textsc{Iterative-Proposal} runs an iterative LLM
proposer with label refinement, and \textsc{Embed-Cluster-Label}
embeds with all-MiniLM-L6-v2, runs k-means at $K{=}16$, names each
cluster with one LLM call, then assigns every item with the cheap
judge (the family popularised by TnT-LLM~\citep{wan2024tntllm};
TnT-LLM's distillation phase is
omitted).\footnote{We did not find a canonical TnT-LLM code release
and the closest third-party reimplementation we surveyed couples to a
prompt-hub service we cannot pin reproducibly, so we use
\textsc{Embed-Cluster-Label} as the family representative.} Two published systems we run from their own repositories,
\textsc{TopicGPT}~\citep{pham2024topicgpt} and Brady \&
Islam~\citep{brady2025iterativetax}, with models served via OpenRouter
to match our routing (implementation details in \S\ref{app:repro}). On
DarkBench, where the gold categories are manipulation types and not
surface topics, \textsc{TopicGPT} also runs with prompts that ask for
the manipulation type and BERTopic with an instruction-tuned embedding
conditioned on it, so each is matched to the target axis.

\paragraph{Setup.}
LLM calls route through OpenRouter on matched models so cost and quality
are comparable; the classical baselines run on a single CPU node, and
wall-clock times reflect this environment. $K{=}16$ for
\textsc{Embed-Cluster-Label} and LDA loosely matches the 20 gold classes
to remove $K$ as a confound.

\paragraph{Metrics.}
We report external metrics (purity, normalized mutual information [NMI], adjusted Rand index [ARI]), intrinsic metrics of
topic quality (NPMI, $C_v$ coherence~\citep{rehurek2010gensim},
label redundancy), cost in \$ from OpenRouter \texttt{usage.cost},
and wall-clock time.
