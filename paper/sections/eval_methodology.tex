\subsection{Evaluation Methodology}
\label{sec:eval-method}

To evaluate whether \textsc{TaxonomyAgent}'s orchestrator--judge loop
produces taxonomies competitive with established baselines while
remaining cheap to run, we run experiments on two benchmarks.

\paragraph{Datasets.}
\textbf{20~Newsgroups}~\citep{lang1995newsgroups} is our classical
benchmark with established gold structure over 20 topic classes. We
use a 487-document stratified subsample (25 per class; 13 documents
are emptied by sklearn's default header/footer/quote filter and
dropped) under five independently sampled random seeds.
\textbf{DarkBench}~\citep{kran2025darkbench} is a real-data benchmark
whose categories are not surface topics: 660 user prompts, each written
to elicit one of six manipulative dark patterns from an AI assistant
(brand bias, user retention, sycophancy, anthropomorphization,
harmful generation, and sneaking), balanced at 110 prompts per
category. Each prompt's category is a property of its manipulative
intent and not of its vocabulary, and the six categories cut across
surface topic, so a taxonomy that groups by topic does not recover
them. We evaluate DarkBench over three seeds with no hint about the
number of categories.

\paragraph{Baselines.}
We compare \textsc{TaxonomyAgent}
(DeepSeek-v4-Flash for both orchestrator and judge) against six
baselines. Two non-LLM classical baselines:
BERTopic~\citep{grootendorst2022bertopic} at library defaults with a
\texttt{min\_topic\_size} sweep, and LDA~\citep{blei2003lda} at
$K{=}16$. Three LLM baselines we wrote to occupy standard design
families: \texttt{single\_shot} prompts one LLM call to both propose
and label, \texttt{iterative\_proposal} runs an iterative LLM
proposer with label refinement, and \texttt{embed\_cluster\_llm}
embeds with all-MiniLM-L6-v2, runs k-means at $K{=}16$, names each
cluster with one LLM call, then assigns every item with the cheap
judge (the family popularised by TnT-LLM~\citep{wan2024tntllm};
TnT-LLM's distillation phase is
omitted).\footnote{We did not find a canonical TnT-LLM code release
and the closest third-party reimplementation we surveyed couples to a
prompt-hub service we cannot pin reproducibly, so we use
\texttt{embed\_cluster\_llm} as the family representative.} Two
concurrent or published systems we run from their own code:
\texttt{topicgpt}~\citep{pham2024topicgpt} (the
\texttt{topicgpt\_python} package with default prompts, routed
through OpenRouter via the OpenAI client's \texttt{base\_url}), and
\texttt{brady\_islam}~\citep{brady2025iterativetax} (their
repository, their prompts, all-MiniLM-L6-v2 + HDBSCAN + iterative
per-cluster topic synthesis), both with the same Llama-3.2-3B
(\texttt{brady\_islam}) or DeepSeek-v4-Flash (\texttt{topicgpt})
model served via OpenRouter to match our uniform OpenRouter routing.
On DarkBench, where the gold categories are manipulation types and not
surface topics, \texttt{topicgpt} also runs with generation and
assignment prompts that ask for the manipulation type, alongside its
default topic prompt, and BERTopic also runs with an instruction-tuned
embedding (multilingual-e5-large-instruct) conditioned on grouping by
manipulation type, alongside all-MiniLM-L6-v2.

\paragraph{Setup.}
LLM calls route through OpenRouter; the classical baselines run on a
single CPU node. Wall-clock times reported in
\S\ref{sec:eval-results-20ng} reflect this environment. All LLM
baselines route through OpenRouter on matched models so that cost
and quality comparisons are apples-to-apples; $K{=}16$ for
\texttt{embed\_cluster\_llm} and LDA matches the 20 gold classes
loosely to remove $K$ as a confound.

\paragraph{Metrics.}
We report external metrics (purity, NMI, ARI), intrinsic metrics of
topic quality (NPMI, $C_v$ coherence~\citep{rehurek2010gensim},
label redundancy), cost in \$ from OpenRouter \texttt{usage.cost},
and wall-clock time.
