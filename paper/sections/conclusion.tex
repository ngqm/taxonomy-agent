\section{Conclusion}
\label{sec:conclusion}

We present an orchestrator/judge agent built on two structural commitments: a typed dispatcher with validate-before-mutate and per-operation failure isolation, and a stateless judge that labels each item against the current taxonomy from item text alone, with no access to the orchestrator's tool-call history. At the install default (DeepSeek-v4-Flash for both roles), the system reaches NMI $0.740 \pm 0.055$ on 20~Newsgroups at $\$0.17$ per run, $1.5\times$ a model-matched embed-cluster-LLM baseline (the family popularised by TnT-LLM), and NMI $0.944 \pm 0.081$ across three seeds on a 149-item synthetic CoT-failure corpus. A Sonnet 4.6 orchestrator at $11\times$ the cost does not improve over the cheap pair on 20NG, so orchestrator quality is not the bottleneck at this scale.
