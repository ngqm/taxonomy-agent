\section*{A\quad Convergence Signal Cross-Check}
\label{app:convergence}

Across the 12 classify events from three-seed 20NG
\textsc{TaxonomyAgent} runs, the don't-fit trajectories are
$[0.00,0.00,0.00,0.00]$,
$[0.05,0.00,0.00,0.00]$, and
$[0.10,0.05,0.00,0.05]$:
the signal is bounded between 0.00 and 0.10, descends quickly toward
the floor in two of three seeds, and shows a non-monotone
$0.00 \to 0.05$ bounce at the end of the third seed. Across the
three terminal $(\text{don't-fit}, \text{purity})$ pairs the Spearman
correlation is $\rho = -0.87$ ($p = 0.33$ at $n = 3$), with the same
sign for NMI and opposite sign for NPMI. The signal saturates at the
floor too quickly to function as a quantitative leading indicator,
so we use it as an operational stopping default in \S\ref{sec:system}
and report external metrics separately.

\section*{C\quad Configuration and Reproducibility}
\label{app:repro}

The numbers in the body were produced from commits pinned in the
repository at \repourl{}, which fixes prompts, hyperparameters, and
seeds. The OpenRouter slugs in the publication runs are
\texttt{deepseek/deepseek-v4-flash},
\texttt{google/gemini-3.1-flash-lite} (the second cheap model in the
robustness rows), and
\texttt{meta-llama/llama-3.2-3b-instruct} (Brady-Islam pipeline); all
calls were routed during 2026-04 through 2026-07. Temperature is the
OpenRouter default for each model and top-$p$ is unconstrained. The
orchestrator is capped at $\texttt{max\_iterations}{=}80$ ReAct steps
and a $\texttt{pool\_limit}$ of $50$ for the judge. DarkBench is the
released \texttt{apart/darkbench} dataset (660 prompts, six balanced
categories); we run it with an empty size hint so the category count
is not supplied.

Baselines: \textsc{Embed-Cluster-Label} embeds with
\texttt{sentence-transformers/all-MiniLM-L6-v2}~\citep{reimers2019sbert} (v3.2.1), runs
sklearn~\citep{pedregosa2011sklearn} k-means at $K{=}16$ with a fixed \texttt{random\_state},
and names each cluster with one DeepSeek-v4-Flash call sampled from
five items. \texttt{lda}~\citep{blei2003lda} uses sklearn
\texttt{LatentDirichletAllocation} at
$K{=}16$, \texttt{learning\_method='batch'}, \texttt{max\_iter=50},
a fixed \texttt{random\_state}, on a $1$-$2$-gram \texttt{CountVectorizer}.
BERTopic v0.16 with library-default UMAP~\citep{mcinnes2018umap}
($n\_neighbors{=}15$, $n\_components{=}5$, $min\_dist{=}0.0$,
a fixed \texttt{random\_state}) and HDBSCAN~\citep{mcinnes2017hdbscan}
($min\_cluster\_size{=}10$, $min\_samples=$None,
\texttt{euclidean}, \texttt{eom}). Brady-Islam reuses the authors'
embedding (\texttt{all-MiniLM-L6-v2}) and HDBSCAN configuration at
\texttt{min\_cluster\_size=5}; a sweep over $\{5,10,15,25,50\}$ did
not raise mean NMI above $0.20$.

Metric definitions: NMI is sklearn's
\texttt{normalized\_mutual\_info\_score} with arithmetic averaging;
ARI is \texttt{adjusted\_rand\_score}; purity is computed by the
Hungarian-optimal one-to-one mapping between discovered and gold
clusters with non-mapped items contributing zero; NPMI uses a
sliding window of $10$ tokens via gensim's \texttt{CoherenceModel};
$C_v$ uses the same window; label redundancy is the fraction of
pairs of discovered categories with cosine similarity above $0.85$
on \texttt{all-MiniLM-L6-v2} embeddings of the label name and
description concatenated.

\section*{E\quad Revise Dispatcher Invariants}
\label{app:dispatcher}
Every operation passed to \texttt{revise\_taxonomy} runs through a pure
handler under two invariants. \emph{Validate before mutate}: each
handler checks every precondition (source category exists, target name
unused, child set non-empty for \textsc{split}, parents disjoint for
\textsc{merge}) before constructing the new taxonomy, so a malformed
\textsc{merge} or \textsc{split} returns the input unchanged with a log
entry the orchestrator reads on the next step. Two early bugs mutated
the working taxonomy before all preconditions had been checked, which
motivated this invariant. \emph{Per-op failure isolation}: the
dispatcher applies operations left to right, recording failures per
operation and continuing with the rest of the batch, so a bad
\textsc{merge} in a three-operation call does not roll back a good
\textsc{add}, and subsequent operations see the post-prefix state.
